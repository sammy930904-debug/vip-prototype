📋 业务规则（核心术语）
  • 电投：会员通过钱包电话/线上下注，由顶代/枪手代为下注的传统模式
  • 转码 / 转码数：开工期间累计有效投注（决定佣金）
  • 上下数：开工净输赢，A 数 + B 数（M/C 双账户）
  • 信贷 / Marker（M）：会员签 M 借款，需在到期前还款，否则计罚息
  • 佣金：基于转码量按占成比 % 结算（即出 / 月结）
  • 星币：转码奖励/服务消费/签到/电投转码累积；到期清空；可兑换或抽奖
  • VIP 等级：VIP（一般户）/ VVIP（股东户）→ auth_type
  • 授权人：会员可授权他人代登录代操作（authorizer）
  • 收款码：会员专属 QR 收款，扫码转账即时到账
  • 订务（memberServices）：订车 / 订房，星币支付 或 存卡支付
  • 交收方案：上下分时选择即出 / 月结 / 公司额度 / 临时额度等
  • 账号体系（Sammy 2026-05-12 拍板）：账房（工作人员）在户口管理里创建会员户口 → 会员用户口号 + 密码直接登录钱包；钱包端无注册入口（不允许会员自助开户）
  • 跨端协作：钱包是 C 端会员自助，账房（账户管理+商品配置）创建户口、配置礼品/抽奖/签到等钱包 APP 内容；代理(B)/总控(S) 不在本端展示
  • 礼品 SKU / 抽奖奖品：由账房 /walletApp/giftManagement + /walletApp/luckyManagement 配置；钱包端只展示，星币消耗值由账房定，钱包不写死
  • USDT 自动收款 (v0.1+)：网关 1:1 自动分配地址(同户口同链永久绑定);链上到账后网关 POST /api/pay/on-chain/notify 推送 → 系统幂等(按 txId)上分;无需上传截图/人工审核;本轮仅支持 TRC20 (TRON 链) × USDT,金额限整数;BSC/ETH/USDC 暂未启用
  • 存币订单号规则 (Sammy 2026-05-13 拍板,方案 A)：STARD2026051300001(18 位)= 业务前缀 STARD(STAR Deposit)+ 日期 YYYYMMDD(8 位)+ 当日序号 NNNNN(5 位 00001~99999,每日 0 点重置);按日聚合直接 LIKE 'STARD20260513%'。提币订单号规则本轮不动(等提币改自动后再定)
  • AML 脏 U 挂账 (v0.2)：网关 /notifyAml(isAML=true) 推送时,该笔 不上分 暂挂账;账房在「USDT 存提币审核 → 存币 Tab → 人工」队列处理 → ① 通过 立即入账 / ② 拒绝 工作人员链上人工退回到会员原转出地址(预计 24-48h);会员可在「⏸ 审核中入款」看挂账金额
  • H5 自动状态更新 (v0.2)：Step 2 页面通过 visibilitychange 事件,用户切到币安/OKX 转账后回到 STAR VIP 时自动 check 一次;后端独立处理 notify,即使用户关掉浏览器,下次打开会通过余额变化+通知中心感知
  • 提币流程 (本轮不变)：仍走 POST /api/pwa/usdtApply/{withdraw,update,cancel} 三步;账房「USDT 存提币审核 → 提币 Tab」人工通过后链上转账;等提币接口文档
