② 入场管理：开工列表 → 开工详情（卖码/回码/担保码/即出/打码/暂存/取暂存）。
③ 场次管理：场次列表 / 客户详情（开台/收工/围数）/ 围数列表。
④ 服务管理：消费列表、礼遇金、积分、星币、消费配置。
⑤ 银头管理：银头概况（含筹码设置/管理）、班次/月度/买码/全部银头、现金/泥码/筹码三种水单。
⑥ 报表管理：会计报表（场次/M报表/月结/转码/上下数/电投佣金）、市场报表（开户/客户/财务/营运）。
⑦ 系统管理：部门/账号/菜单/系统配置（场馆/VIP/币种/汇率/积分/信贷/加密支付配置 🆕/常量）、操作日志。
⑧ 消费中心 / 通讯管理：消费服务、短信群发/模板/渠道/历史、电话坐席/通话记录。
⑨ 钱包 App 管理：钱包 H5 后台 → 签到/礼品/抽奖/公告/消息/客服热线/审核/链上交易记录 🆕/可疑入款审查 🆕/黑名单/APP 日志。
📋 业务规则
① 账号体系：账房创建 客户户口 → 钱包端无注册入口，会员用户口号 + 密码登录；密码可在账房"账户详情 → 重置钱包登录密码"重置（注：移除硬编码 star99，展示 API 实际密码）。
② 5 端关系：
 • C 端 VIP电投（会员 H5）
 • B 端 电投代理（代理 H5+PC）
 • S 端 电投总控（运营 PC）
 • 钱包端（会员 H5）
 • 账房端（本系统，工作人员 PC）
③ 业务术语：电投/转码量/上下数/银头/泥码/现金码/工作码/M报表 (Maker)/水单/围数/月结/打赏 (礼遇金)/星币/占成/顶代/枪手。
④ 关键流程：入场 → 开工（开台/卖码/担保码/即出）→ 转码 → 上下数 → 收工 → 银头水单 → 月结 → 交收。
⑤ 权限：ButtonPower 按钮级权限；按 部门 + 职务 + 职级 + 角色 4 维组织匹配菜单/操作权限。
⑥ 跨端协作：
 • 公司账户 ↔ 电投代理后台（B 端代理为下游分发）
 • 银头水单 ↔ 总控 S 端桌台运营数据
 • 钱包 App 配置 → 钱包端 H5 实时拉取
⑦ USDT 收款流程 (v0.10 — 2026-05-14 改)：会员钱包端「快捷存币」不输入金额、不上传截图;仅支持 TRC20 (TRON 链) × USDT(BSC/ETH/USDC 暂未启用);系统调网关 POST /api/third/pay/save 给该户口生成专属 TRC20 地址(永久绑定 1:1);链上到账 → 网关 POST /api/pay/on-chain/notify(HMAC-SHA256 签名)推 webhook → 商户按 txId 幂等;所有 U 一律自动入账(取消人工审核/退回流程)。
  💰 入账金额规则(核心):一律按链上 notify 推送的 amount 入账(链上为准);会员钱包端不再输入金额,转多少入多少;场馆 = OKADA(本轮所有 USDT 存款固定);存卡交易流水的"交易金额"=链上实际入款,"存卡金额"=入账后余额。
  ⚠️ 风险标记规则:网关 notify 推送 isAML 布尔字段(true=高风险地址/false=干净) — 接口暂不返回风险等级/类型;不阻断入账,仅在账房列表/详情上以"⚠️ 高风险地址"标记展示,供风控部门事后追查;若后续需要等级/标签,需让网关下版本扩展 notify 字段。
  存币订单号规则:STARD2026051400001(18 位)= STARD(STAR Deposit)+ YYYYMMDD + NNNNN(当日序号 00001-99999,每日 0 点重置);提币订单号本轮不动,等提币改自动后再定。
⑧ AML 处理 (v0.10 简化):不再人工审核,不再退回;命中 AML(isAML=true)的入款也自动入账,系统在该笔流水上打 ⚠️ 高风险地址标记(接口只返回布尔,无等级);事后由风控部门基于地址维度累计脏 U 笔数追溯/冻结(地址 ≥3 笔脏 U 自动建议禁用),账房端只查看不处理。
⑨ USDT 提币流程 (本轮不变)：仍走 POST /api/pwa/usdtApply/{withdraw,update,cancel} → 「walletApp/review/withdrawal → 提币 Tab」人工审核 → 通过后链上转账(等提币接口文档后再改自动)。
